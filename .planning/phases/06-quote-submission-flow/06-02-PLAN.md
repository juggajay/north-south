**Design decision (from CONTEXT.md):**
- User is already logged in (auth required)
- Design is already auto-saved (from Phase 04)
- Need to either: (a) get existing designId from auto-save, or (b) create design on submission

**Recommended approach:**
- Check if there's a currentDesignId in a store or context
- If not, create the design on-the-fly when user clicks "Submit for Quote":
  1. Call designs.create mutation with current config from useCabinetStore
  2. Use returned designId for submission.create

This keeps the flow simple and doesn't require changes to auto-save logic.

**Implementation:**
```typescript
// In StepReview.tsx
const [showSubmission, setShowSubmission] = useState(false);
const [designId, setDesignId] = useState<string | null>(null);
const createDesign = useMutation(api.designs.create);
const { user } = useAuth();
const config = useCabinetStore((state) => state.config);

const handleStartSubmission = async () => {
  // Create design if not already created
  const id = await createDesign({
    userId: user?._id,
    productType: "pantry", // or get from context
    config: serializeConfig(config), // serialize Map to plain object
  });
  setDesignId(id);
  setShowSubmission(true);
};

// Helper to serialize Map to plain object for Convex
function serializeConfig(config: CabinetConfig) {
  return {
    dimensions: config.dimensions,
    slots: Object.fromEntries(config.slots),
    finishes: config.finishes,
  };
}
```
